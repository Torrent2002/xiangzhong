小红书社交突破 · 隐含意图驱动的相亲匹配
演示版 v0.1
PROBLEM
相亲软件，听不懂"说不清的喜欢"
现状两条路，都有天花板
1滑照片：被动、低效、纯靠肉眼，无法主动筛选
2填字段匹配：字段是平台预设的，覆盖不到"瓜子脸、不戴眼镜、书卷气"
一个真实场景
小 A 想找"不戴眼镜、温柔、有点文艺"的女生
Tinder 滑了 200 张没找到 → 在 Soul 填了 20 个字段也表达不出来 → 流失
不是体验问题，是表达 + 理解能力的缺位。
MECHANISM
偏好有两类，AI 各自击破
A 显性偏好用户已明确
用户自己已明确的偏好
例：不戴眼镜、瓜子脸、温柔文艺
获取：自然语言描述 / 参考图 / 预设字段
AI 作用：解析为意图向量
B 隐性偏好用户自己还没发现
用户自己还没发现的偏好
例：嘴上说喜欢文艺，实际心动更多是高冷运动风
获取：停留时长 / like·pass / 反复回看 等行为信号
AI 作用：反推为偏好向量

显性偏好让用户上手，隐性偏好让产品越用越懂
DECISIONS
四个关键取舍
D1为什么独立 app
场景聚焦、合规隔离、不污染小红书内容生态
D2冷启动
借小红书公开笔记（授权）做初始 profile 池；先做单边工具（"帮你找到理想型的长相参考"）切入，再过渡双边匹配
D3匹配透明化 + 行为闭环
主动展示匹配理由（建信任）；同时采集 like/pass/停留 反推隐性偏好，让推荐越用越准
D4价值观护栏
做"品味匹配"而非"外貌淘汰"，不展示被淘汰者，不做单一颜值打分排名
RISKS
知道边界在哪，才做得下去
AI 误判 / 幻觉
→
匹配理由透明 + 用户反馈闭环 + 人工抽审
外貌歧视 / 物化
→
包装为"品味匹配"；种族、残疾等不可作为筛选条件
隐私合规
→
照片授权、不存原始生物特征数据、可一键删除；行为反推只用聚合信号，不画像到具体个人
用户信任
→
灰度上线，先做低 stakes 推荐；隐性偏好提示要可解释、可关闭
MVP & NEXT
先做小，再做大

MVP 只做

  • 显性偏好：文字描述 + 参考图 双通道匹配
  • 隐性偏好：行为信号采集 + 基础反推（先做 like/pass 统计）
  • 单城市 / 单人群灰度

MVP 不做（克制）

  • 视频、语音、复杂社交关系

成功指标

  • 匹配准确率（用户 like 率）
  • 描述 → 首次心动 时长
  • 次日 / 7 日留存
