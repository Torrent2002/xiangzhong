# PPT Agent 需求文档

> 本文档是给"做 PPT 的 agent"看的执行 brief。你的任务是把它变成一个**可全屏演示、离线可用、可导出 PDF** 的 HTML 幻灯片。

---

## 1. 背景与目标

- **场景**：30 分钟内要向面试官呈现一个产品方案。
- **产品一句话**：在小红书之外新起的相亲平台「相中」，用 AI 理解用户"说不清的喜欢"（隐含意图），把传统相亲软件从"填字段检索"升级为"语义 + 视觉向量匹配 + 行为反推"。
- **Tagline（全 deck 反复出现的口号）**：**"你说不清的喜欢，AI 帮你找得到。"**
- **核心产品概念（本 deck 的主线）**：把用户偏好分两类，AI 各自击破——
  - **A 显性偏好**：用户自己已明确的（"不戴眼镜、瓜子脸"）→ 直接用预设 / 自然语言 / 参考图获取，AI 解析为意图向量。
  - **B 隐性偏好**：用户自己还没发现的（嘴上说喜欢文艺，实际心动更多是高冷运动风）→ 靠行为信号（停留 / like·pass / 反复回看）反推，AI 推断为偏好向量。
- **受众**：产品/技术面试官。视觉要专业、克制、有产品感，而不是花哨。

## 2. 交付物规格

- **形式**：单个、自包含的 `index.html` 文件（所有 CSS/JS 内联，**不依赖任何 CDN 或网络**——演示现场可能断网）。
- **幻灯片框架**：可自选实现方式（手写 CSS + 翻页 JS，或内联 reveal.js）。优先保证**离线可播 + 可导出 PDF**（浏览器打印为 PDF 即可）。
- **页数**：**7 页**，不多不少。每页一个核心观点，禁止文字堆砌。
- **语言**：简体中文。
- **字体**：系统中文字体栈 `PingFang SC, "Helvetica Neue", Arial, sans-serif`。
- **尺寸**：默认 16:9，全屏播放要正常。
- **输出文件**：`deck/index.html`（放在当前目录下的 `deck/` 文件夹）。

## 3. 视觉方向

- **风格**：简洁、克制、产品发布会感（参考苹果 Keynote / 小红书官方视觉的留白与呼吸感）。
- **配色**：
  - 主点缀色：小红书红 `#FF2442`（只用于强调，不要大面积铺）
  - 文字：`#1a1a1a`
  - 背景：`#ffffff` / 浅灰 `#f7f7f7` 交替
  - 辅助灰：`#999` 用于次要文字
  - 两类偏好可用色块区分：显性偏好用浅蓝 `#E8F0FE` 底，隐性偏好用浅红 `#FDECEE` 底（仅 P2 用，别滥用）
- **留白要大**，每页信息密度低，靠大标题 + 少量要点撑起。
- **关键页必须有图**：
  - P2 机制页：画一张 pipeline 示意图（见下文），不要只有字。
  - P3 旅程页：画 **4 个手机线框图**（纯 CSS 画的手机外框 + 界面内容即可，不需要真实截图）。
- 禁止用 emoji 当主要视觉元素；可以用极少量 ✓ 符号表对勾。

## 4. 逐页内容（文案已定稿，照抄，不要改写含义）

### P0 · 封面
- 大标题：**相中**
- 副标题：**你说不清的喜欢，AI 帮你找得到。**
- 小字：小红书社交突破 · 隐含意图驱动的相亲匹配
- 底部小字：演示版 v0.1

### P1 · 问题：长尾需求为什么没人满足
- 大标题：**相亲软件，听不懂"说不清的喜欢"**
- 左栏「现状两条路，都有天花板」：
  - **① 滑照片**：被动、低效、纯靠肉眼，无法主动筛选
  - **② 填字段匹配**：字段是平台预设的，覆盖不到"瓜子脸、不戴眼镜、书卷气"
- 右栏「一个真实场景」：
  - 小 A 想找"不戴眼镜、温柔、有点文艺"的女生
  - Tinder 滑了 200 张没找到 → 在 Soul 填了 20 个字段也表达不出来 → 流失
- 底部结论（红字）：**不是体验问题，是表达 + 理解能力的缺位。**

### P2 · 突破点：把偏好分两类，用 AI 各自击破（必须有图）
- 大标题：**偏好有两类，AI 各自击破**
- 左右两栏（色块区分），每栏 标题 + 一句定义 + 获取方式 + AI 作用：
  - **A 显性偏好（用户已明确）**
    - 例：不戴眼镜、瓜子脸、温柔文艺
    - 获取：自然语言描述 / 参考图 / 预设字段
    - AI 作用：解析为 **意图向量**
  - **B 隐性偏好（用户自己还没发现）**
    - 例：嘴上说喜欢文艺，实际心动更多是高冷运动风
    - 获取：停留时长 / like·pass / 反复回看 等行为信号
    - AI 作用：反推为 **偏好向量**
- 下方 pipeline 图，横向流程：两条向量汇入 → 与候选特征向量匹配 → 打分排序 → Top-N 推荐 → 反馈微调（反馈同时回流更新两类偏好）
- 底部一行（红字）：**范式升级：标签检索（schema 匹配）→ 语义 + 视觉向量匹配 + 行为反推**

### P3 · 核心体验 / 用户旅程（画 4 个手机线框图）
- 大标题：**从"你描述" 到 "AI 比你更懂你"**
- 横向 4 个手机线框，每个下方一句话说明：
  1. **输入（显性偏好）**：输入框 `温柔、不戴眼镜、文艺、瓜子脸` + 按钮「AI 帮我找」
  2. **AI 拆解（透明化）**：展示 AI 把描述拆成的维度：
     - 气质：温柔 ✓ ｜ 眼镜：无 ✓ ｜ 风格：文艺 ✓ ｜ 脸型：瓜子脸 ✓
  3. **推荐 + 采集行为**：一张候选卡片 + "为什么推荐给你"：
     - `不戴眼镜 ✓ ｜ 瓜子脸 ✓ ｜ 文艺风 ✓ ｜ 唯一差异：她偏活泼`
     - 卡片底部 ♡ 心动 / ✕ 跳过（注明：行为信号被记录）
  4. **几天后：AI 发现隐性偏好**：一张提示卡：
     - `你嘴上说喜欢温柔文艺，但最近心动更多是高冷运动风——要不要看看？`
     - 底部按钮：「看看」/「不是，继续按原来的」
- 底部一行：**显性偏好让用户上手，隐性偏好让产品越用越懂**

### P4 · 关键产品决策（体现判断力）
- 大标题：**四个关键取舍**
- 四个卡片，每个 标题 + 一句话：
  - **D1 为什么独立 app**：场景聚焦、合规隔离、不污染小红书内容生态
  - **D2 冷启动**：借小红书公开笔记（授权）做初始 profile 池；先做单边工具（"帮你找到理想型的长相参考"）切入，再过渡双边匹配
  - **D3 匹配透明化 + 行为闭环**：主动展示匹配理由（建信任）；同时采集 like/pass/停留 反推隐性偏好，让推荐越用越准
  - **D4 价值观护栏**：做"品味匹配"而非"外貌淘汰"，不展示被淘汰者，不做单一颜值打分排名

### P5 · 风险与边界
- 大标题：**知道边界在哪，才做得下去**
- 四条，每条 = 风险 + 应对：
  - **AI 误判 / 幻觉** → 匹配理由透明 + 用户反馈闭环 + 人工抽审
  - **外貌歧视 / 物化** → 包装为"品味匹配"；种族、残疾等不可作为筛选条件
  - **隐私合规** → 照片授权、不存原始生物特征数据、可一键删除；行为反推只用聚合信号，不画像到具体个人
  - **用户信任** → 灰度上线，先做低 stakes 推荐；隐性偏好提示要可解释、可关闭

### P6 · MVP 范围 + Next Steps
- 大标题：**先做小，再做大**
- 左栏「MVP 只做」：
  - 显性偏好：文字描述 + 参考图 双通道匹配
  - 隐性偏好：行为信号采集 + 基础反推（先做 like/pass 统计）
  - 单城市 / 单人群灰度
- 左栏「MVP 不做（克制）」：
  - 视频、语音、复杂社交关系
- 右栏「成功指标」：
  - 匹配准确率（用户 like 率）
  - 描述 → 首次心动 时长
  - 次日 / 7 日留存
- 底部 Roadmap 横条：**单点工具 → 双边匹配 → 关系沉淀**
- 末尾小字：谢谢，欢迎提问

## 5. 技术约束

- **必须离线可用**：不要 `<link>` 外部字体、不要 CDN、不要外部图片。所有图标用内联 SVG 或 CSS 画。
- 单文件 `index.html`，双击即可在浏览器打开。
- 翻页交互：左右方向键 / 空格 翻页；右上角显示 `当前页/总页`。
- 提供一个"导出 PDF"的友好提示（页面角落小字："浏览器 Cmd+P → 存为 PDF"）。
- 代码注释保持简洁，不要在页面里露出调试信息。
